# BossAgent 业务逻辑

本文件定义了 CLI、FastAPI 与油猴脚本共享的业务规则。用户可通过 `.bat` 启动本地服务和 CLI；油猴脚本只负责浏览器页面读取与执行，最终是否进入打招呼由后端决策。

## 1. 配置与准备

1. 配置模型、评分阈值、每轮联系上限、发送方式。
2. 保存或上传简历；PDF 会提取为 Markdown，并可渲染首张简历图片。
3. 由模型生成或手动维护用户画像、搜索标签和打招呼话术。
4. `start` 前必须有搜索标签、用户画像和已确认话术；缺少时 CLI 会提示补齐。

## 2. 职位处理主流程

```text
BOSS 搜索页 → 油猴读取职位详情 → 后端评分前决策
  ├─ 已沟通 / 配置筛选 / 历史去重 / 公司联系上限 → 记录 skip
  └─ 通过 → LLM 语义评分 → 达阈值 greet / 未达阈值 skip
greet → 打开聊天页 → 发送已确认话术（可附简历图） → 写入历史
```

## 3. 评分前决策优先级

由 `decision_service.evaluate_preflight` 统一执行，优先级从高到低：

1. 页面显示已经沟通：永远跳过，并记为 `already_contacted`。
2. 配置筛选：正式/实习、城市、职位关键词、屏蔽公司、薪资范围。
3. 历史去重：已打招呼或 24 小时内已处理的职位。
4. 公司上限：达到 `max_contacts_per_company` 时跳过。
5. 只有以上都通过，职位才会调用 LLM 评分。

关闭 `skip_contacted_companies` 时，仅第 3、4 条失效；页面明确显示“已沟通”仍会跳过。

## 4. LLM 评分与语义岗位意图

评分基于学历专业（15%）、技术栈（50%）与项目经验（35%）。`job_filter_target_roles` 是语义门槛，不是标题字面匹配：例如目标为“Agent开发”时，“AI训练师、数据标注、模型训练、模型运营”即使出现 AI 关键词，也应判为不同岗位方向并获得低分。

模型输出不合规或失败时，系统记录 `model_failed_skip`，不会用不可靠分数发送打招呼。

## 5. 执行、状态与历史

- 油猴仅在后端控制状态为 `running` 时执行；`pause` 和 `stop` 会停止后续页面操作。
- `session_greet_limit` 控制单轮成功打招呼次数，默认 50。
- SQLite 保存职位、动作、事件、最终动作和运行批次；`history` 与 `report` 用于追溯。
- 遇到登录失效、验证码、风控、页面元素异常或人工中断时，脚本暂停并写入事件，等待人工处理。
